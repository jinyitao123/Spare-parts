## 一、全局设计语言

### 1.1 设计基调

工业管理软件，不是消费级App。要求：**信息密度高、操作效率优先、视觉干净不花哨**。

色彩体系：
- 主色：深蓝 `#1B2A4A`（导航栏、标题）
- 辅助色：钢蓝 `#4A90D9`（可交互元素、链接、Agent标识）
- 成功/安全：`#22C55E`
- 警告：`#F59E0B`
- 危险/异常：`#EF4444`
- 呆滞：`#8B5CF6`（紫色，区别于红色的"危险"）
- 背景：`#F8FAFC`（浅灰白）
- 卡片：`#FFFFFF`
- 文字：`#1E293B`（主文字）、`#64748B`（次要文字）

字体：系统默认中文字体（思源黑体 / 苹方 / 微软雅黑），不用花哨字体。

Agent相关元素统一用 `#4A90D9` 钢蓝色标识 + ✦ 符号，让用户能一眼区分"这是AI说的"和"这是系统数据"。

### 1.3 四套角色视图

不同角色登录后看到的导航菜单不同：

| 角色 | 可见菜单 | 首页内容 |
|---|---|---|
| 设备工程师 | 工作台、备件库房 | 待补全记录、常用备件库存、Agent快捷入口 |
| 库管员 | 工作台、备件台账、备件库房、呆滞管理 | 今日出入库汇总、盘点提醒、库存异常 |

| 管理层 | 工作台、看板 | 纯数字看板 |

**左侧：库存列表**

标准表格视图，列包括：
- 物料编码
- 备件名称
- 规格型号
- 当前库存（数字 + 状态图标：✅充足 / ⚠️临界 / 🔴不足 / 🟣呆滞）
- 安全库存
- 库存金额
- 末次变动日期
- 本月领用次数

表格行颜色提示：
- 正常行：白底
- 低于安全库存：浅红底 `#FEF2F2`
- 呆滞备件：浅紫底 `#F5F3FF`

顶部操作栏：
- [出库] 按钮→打开出库表单（通道B扫码模式下使用）
- [入库] 按钮→打开入库表单
- [退库] 按钮→打开退库表单
- [导出] 按钮→导出Excel（客户明确需求：分析备件用在了哪些地方、金额多少）

**右侧：Agent-1 对话区**

这是 Agent-1 的主战场。占页面右侧约 40% 宽度。

对话区的交互：
- 工程师输入"3号线电机要换轴承"
- Agent 回复包含：合理性校验结果、推荐方案（含替代件和呆滞件）、生成的领用指令
- 领用指令以**操作卡片**形式出现在对话流中，卡片上有 [确认领用] 按钮
- 点击确认后，左侧库存列表实时更新数量

## 九、移动端适配要点

工程师在车间里用手机/平板操作的场景很多，关键页面需要移动端适配。

### 9.1 移动端优先的页面

| 页面 | 移动端形态 | 说明 |
|---|---|---|
| 工程师工作台 | 全屏卡片流 | 快捷领料入口 + 待补全记录 + 常用备件 |
| 库房出库 | 全屏表单 | 扫码 → 填数量 → 选原因 → 确认，4步完成 |
| Agent对话 | 全屏聊天界面 | 类似微信对话，操作卡片内嵌 |
| 通道B补全 | 推送通知 + 内嵌快捷回复 | 通知栏直接点选，不用打开App |

### 9.2 移动端不需要的页面

看板、台账详情、采购审批详情——这些在电脑上操作即可。
