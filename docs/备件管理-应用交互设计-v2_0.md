# 备件管理智能平台 · 应用交互设计

**永祥科技（通威集团灯塔工厂）| inocube 智能平台**

Application Interaction Design — AI Native

v2.0 · 2026.03

---

> *核心设计原则：Agent是叙事者，不是助手。*
> *用户不是在"操作系统+偶尔问AI"，而是在"跟AI对话+AI渲染出可操作的界面"。*
> *传统表格和表单是AI输出中嵌入的组件，不是独立的页面。*

---

## 第一章 设计哲学

### 1.1 从"页面+AI助手"到"AI画布+数据组件"

| 维度 | 传统模式（v1.0） | AI Native模式（v2.0） |
|---|---|---|
| 主界面 | 表格/表单（固定布局） | Agent对话流（动态生成） |
| AI的位置 | 右侧对话框（配角） | 整个页面（主角） |
| 用户视线 | 左看表格 → 右看AI → 来回切换 | 从上往下阅读Agent的叙事 |
| 信息获取 | 用户自己从表格中发现异常 | Agent主动告诉你异常在哪 |
| 操作方式 | 点按钮、填表单 | 在对话流中点击嵌入式操作卡片 |
| 数据可视化 | 固定位置的图表 | Agent回复中动态嵌入的图表 |
| 叙事者 | 数据（用户自己读） | Agent（帮你读并讲出来） |

### 1.2 不是所有场景都适合对话流

AI Native不是"一切皆对话"。有些场景对话流是最优的，有些场景结构化界面更高效。

| 场景类型 | 交互形态 | 原因 |
|---|---|---|
| 决策辅助（领用建议、采购审批、呆滞处置） | **AI对话流** | 需要分析、推理、权衡——Agent的长项 |
| 全局概览（金额看板、目标追踪） | **AI叙事+嵌入图表** | 需要Agent解读数字背后的含义 |
| 高频录入（盘点核对） | **传统表格** | 逐行核对数据，对话流反而低效 |
| 批量浏览（查看全部库存明细） | **AI可展开的嵌入式表格** | Agent先做摘要，需要明细时展开表格 |

```
│          │                                                    │
│  ─────── │                                                    │
│  📋 盘点  │                                                    │

│          │                                                    │
└──────────┴───────────────────────────────────────────────────┘
```

**左侧导航栏：** 保留，但功能重新定义——不是"进入某个功能页面"，而是"切换Agent的上下文"。点击"库房"不是打开一个库存表格页，而是唤起Agent-1（库存管家）并加载库房相关上下文。点击"采购"唤起Agent-5（采购建议师）。

**导航栏分两组：**
- 上组（AI驱动）：工作台、库房、采购、呆滞、驾驶舱——每个都是AI对话流
- 下组（传统视图）：盘点、台账——保留结构化表格界面

**右侧AI画布：** 占据主要空间。所有AI对话流和渲染的组件都在这里。没有"左表格右聊天"的分栏。

### 2.2 四套角色视图

| 角色 | 可见导航项 | 默认Agent | 首次打开看到什么 |
|---|---|---|---|
| 设备工程师 | 工作台、库房 | Agent-1 库存管家 | 今日简报：待补全记录+常用备件状态+快捷领料入口 |
| 库管员 | 工作台、库房、呆滞、盘点、台账 | Agent-1 库存管家 | 今日简报：出入库汇总+库存异常+盘点提醒 |
| 工段长 | 全部 | Agent-10 策略顾问 | 驾驶舱：目标达成+路径+待审批+决策回溯 |
| 管理层 | 工作台、驾驶舱 | Agent-10 策略顾问 | 驾驶舱：库存结构透视+达标进度+趋势 |

### 2.3 Agent渲染组件库

Agent的每一次回复可以包含以下组件。组件库分三个优先级落地，并支持后续扩展。

#### P0 组件（Day-1 必须）

| 组件 | 用途 | 形态 |
|---|---|---|
| **文字段落** | Agent的分析和解释 | Markdown渲染，支持加粗、✦标记 |
| **数据卡片** | 单个备件/头寸/设备的关键信息 | 紧凑的信息面板，标题+关键指标+状态标签 |
| **操作按钮组** | 确认/驳回/选择方案 | 按钮排列，支持主操作+次操作样式 |
| **快捷选项** | 点选式回答（替代打字） | 胶囊按钮，点击即发送 |
| **状态条** | 库存水位 | 水平进度条，当前值 vs 安全线，颜色标注 |
| **提醒横幅** | 异常/警告/建议 | 带图标的彩色横幅（红/黄/绿/紫） |

#### P1 组件（首月内）

| 组件 | 用途 | 形态 |
|---|---|---|
| **嵌入式表格** | 可排序可筛选的数据列表 | 对话流中内嵌，默认显示5行+可展开 |
| **柱状图** | 消耗对比、领用Top3 | 嵌入式echarts/recharts图表 |
| **折线图** | 趋势（金额趋势、消耗趋势） | 嵌入式图表，支持多线叠加 |
| **堆叠条形图** | 库存结构四象限 | 横向堆叠，颜色区分 |
| **展开/收起面板** | 详情下钻 | 点击展开更多信息，收起后不占空间 |

#### P2 组件（3个月内）

| 组件 | 用途 | 形态 |
|---|---|---|
| **对比面板** | 方案A vs 方案B | 左右并排，差异项高亮 |
| **推演滑块** | 参数调整+实时计算 | 滑块控件+实时刷新的结果区域 |
| **时间轴** | 达标路径、采购进度 | 横向时间轴，节点可点击 |
| **环形图** | 占比分析 | 适用于呆滞分类占比、金额构成 |
| **分组卡片流** | 采购建议分优先级展示 | 卡片按优先级排列，高优展开低优折叠 |

#### 后续扩展组件（按需增加）

| 组件 | 场景 | 引入条件 |
|---|---|---|
| 关系图谱 | 设备-备件-头寸关联可视化 | 图谱层上线后 |
| 热力图 | 各库房各类备件的库存健康度 | 数据积累6个月后 |
| 甘特图 | 采购时间线和保养排期 | Agent-6上线后 |
| 桑基图 | 资金流向（采购→库存→消耗→哪台设备） | 完整本体阶段 |
| 地图/平面图 | 库房布局和库位导航 | 如果需要库位级管理 |

**扩展机制：** 新组件通过组件注册表添加，Agent的提示词中声明可用组件列表。新增一个组件不需要修改Agent逻辑，只需注册组件+告诉Agent"你现在多了一种可以渲染的组件"。

- 每个模块自带可视化组件（进度条、趋势图、堆叠条）和操作入口

工程师在通知栏直接点选，不需要打开App。

## 第十章 通知系统

Agent主动推送的通知是整个系统的"心跳"——让用户即使不打开系统也能感知到重要事件。

### 10.1 通知类型

| 通知级别 | 触发Agent | 场景 | 形式 |
|---|---|---|---|
| 🔴 紧急 | Agent-1 | 安全库存见底、高价值备件异常领用 | 即时推送+弹窗 |
| 🔴 紧急 | Agent-5 | 采购审批即将到期 | 即时推送 |
| 🟡 注意 | Agent-1 | 频次异常检测 | 即时推送 |
| 🟡 注意 | Agent-1 | 通道B待补全提醒（15分钟后） | 定时推送 |
| 🟢 建议 | Agent-9a | 今日优化建议已生成 | 每日17:00推送 |
| 🟢 建议 | Agent-10 | 月度达标路径更新 | 每月1号推送 |
| 🟣 信息 | Agent-4 | 新增呆滞件 | 每月1号推送 |
| 🟣 信息 | Agent-10 | 决策回溯报告 | 每月1号推送 |

### 10.2 通知与对话的衔接

每条通知都可以"展开为对话"——用户点击通知后，直接进入相关Agent的对话流，Agent自动加载相关上下文。

```
通知：🟡 3号线电机本月第4次更换轴承
  ↓ 用户点击
进入 Agent-1 对话流，Agent自动展示：
  · 消耗趋势图
  · 异常分析
  · 建议操作（排查/继续领用/升级维修方案）
```

## 第十二章 与 v1.0 的核心差异

| 维度 | v1.0 设计 | v2.0 设计 |
