## 第一章 设计基调

### 1.1 气质定位

这个产品不是一个冰冷的ERP系统，也不是一个花哨的消费App。它是一个**有温度的智能工具**——像一间光线温暖的工坊，桌面上摊着图纸和数据，你身边坐着一个懂行的参谋，他帮你看数据、讲分析、给建议，你做决策。

三个关键词：

**温暖** —— 暖色调底色、自然色系、没有冷硬的机械感。用户长时间使用不疲惫。

**呼吸** —— 大量留白、内容不拥挤、Agent回复有从容的节奏感。每个信息模块之间有充足的呼吸空间。

**从容** —— 不急不躁。Agent不会用感叹号和红色大字吓你。即使是异常提醒，也是平静地陈述事实、给出建议。整个界面的"说话音量"是对话级的，不是广播级的。

### 1.2 设计检验标准

拿到任何一个页面设计稿，问三个问题：

1. **亚麻布测试**：截图后去色为灰度，整体感觉是温暖的还是冰冷的？如果冰冷，说明色调偏蓝灰了。
2. **呼吸测试**：眯眼看页面，信息模块之间的留白是否让你觉得舒服？如果觉得拥挤，增加间距。
3. **音量测试**：页面上最大最亮的元素是什么？如果是导航栏或按钮（而不是内容），说明"结构比内容吵"。

## 第二章 色彩体系

### 2.1 底色（Ground）—— 亚麻布的温度

继承 Weave 的核心理念：**背景永远不是纯白**。默认画布是温暖的米白色，像没有漂白的亚麻布。

| 名称 | 色值 | 用途 |
|---|---|---|
| 亚麻白 | `#FFFFFF` | 悬浮卡片、输入框、Agent组件卡片——需要从底色上"浮起来"的元素 |
| 生麻色 | `#F7F5F2` | **页面默认底色**——所有页面的背景 |
| 浅麻色 | `#EFECEA` | 内嵌区域、次级背景——如展开的详情面板背景 |
| 亚麻灰 | `#A09A94` | 占位文字、时间戳、辅助信息 |
| 泥土色 | `#6B6560` | 次要文字——描述、标签 |
| 胡桃色 | `#1A1A1E` | **主文字**——暖调深色，不是纯黑 |

**禁止：** `#000000`（纯黑）、`#FFFFFF`用作页面背景、任何蓝灰色调的灰色。

### 2.2 点缀色（Pigment）—— 赭陶一抹

整个界面只用一个点缀色：**赭陶色** `#D4714A`——烧窑陶器的颜色。它出现在：

- 品牌标识
- 每个页面最多一个主操作按钮
- 当前激活的导航项标识
- Agent的 ✦ 符号

**点缀预算：每个视口内最多2个赭陶色元素。** 超过2个就减掉最不重要的。

| 名称 | 色值 | 用途 |
|---|---|---|
| 赭陶 | `#D4714A` | 主按钮填充、品牌标识、✦ 符号 |
| 赭陶浅 | `#F5DDD0` | 主按钮悬停态背景 |
| 赭陶深 | `#A3563A` | 主按钮按下态 |

### 2.3 信号色（Signal）—— 自然染料

状态信号用自然色系，不用霓虹色。每个信号色有两个变体：底色（用于背景填充）和文字色（用于文字和图标）。

| 名称 | 底色 | 文字色 | 含义 | 用于 |
|---|---|---|---|---|
| 鼠尾草绿 | `#E8F5E8` | `#2D6A2D` | 安全、充足、正常 | 库存充足、审批通过、同步成功 |
| 赭黄 | `#FFF3E0` | `#8B5E0A` | 注意、临界、接近阈值 | 库存临界、频次偏高、待处理 |
| 茜红 | `#FCEBEB` | `#9B2C2C` | 异常、不足、紧急 | 库存不足、异常拦截、同步失败 |
| 靛蓝 | `#E8F0FE` | `#1A56A0` | 信息、Agent标识、链接 | 暂不在本场景使用，预留 |
| 紫藤 | `#F3EFFE` | `#6B4DC4` | 呆滞 | 呆滞备件标记——独立于"异常"的语义 |

**信号预算：每张卡片内最多2种信号色。** 如果一张卡片同时出现绿、黄、红，说明信息过载——简化状态层级。

### 2.4 Agent区分色 —— 微妙的布纹

Agent的回复区域与用户消息、系统数据之间用**微妙的背景色差异**区分，不用强烈的边框或色块。

| 区域 | 背景色 | 说明 |
|---|---|---|
| 页面底色 | `#F7F5F2`（生麻色） | 整体背景 |
| Agent回复区域 | `#F2F0EC`（比底色深一级，差异约3%） | Agent的叙事段落和分析文字 |
| Agent组件卡片 | `#FFFFFF`（亚麻白） | 数据卡片、图表、操作按钮组——从Agent回复区域上浮起 |
| 用户消息 | `#FFFFFF`（亚麻白）右对齐 | 用户输入的文字 |

这个差异非常微妙——不是"AI区域是蓝色的、数据区域是白色的"，而是Agent说话的区域像是铺了一层薄薄的亚麻纱，比旁边的底色深了一点点。用户不会意识到"哦这里颜色不同"，但潜意识会感觉到"这段内容的性质跟那段不太一样"。

**✦ 符号：** Agent生成的所有内容开头用赭陶色的 ✦ 标记。这是唯一的显式AI标识——不是大字"AI分析"，而是一个小小的、温暖的符号。

## 第三章 字体

### 3.1 字体选择

中文：**思源黑体（Source Han Sans / Noto Sans CJK）**——如果系统没有安装，降级到苹方/微软雅黑/系统默认。不使用宋体（太正式）、不使用圆体（太随意）。

西文和数字：**Inter**——如果可用。Inter的数字等宽特性对金额和库存数量展示很有帮助。降级到系统Sans-serif。

代码和编码：**JetBrains Mono**——用于物料编码、系统日志。降级到等宽系统字体。

```css
--font-sans: "Source Han Sans", "Noto Sans CJK SC", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
--font-mono: "JetBrains Mono", "SF Mono", "Fira Code", "Consolas", monospace;
```

### 3.2 字号体系

| 名称 | 字号 | 字重 | 行高 | 用途 |
|---|---|---|---|---|
| 页面标题 | 20px | 500 | 1.4 | 页面级标题："库存管家""采购建议师" |
| 区块标题 | 16px | 500 | 1.4 | Agent叙事中的小节标题、卡片标题 |
| 正文 | 14px | 400 | 1.75 | **所有正文**——Agent回复、描述、标签。行高1.75给阅读充足呼吸 |
| 数据 | 14px | 500 | 1.4 | 表格中的数值、金额——用500字重让数字更醒目 |
| 大数字 | 28px | 500 | 1.2 | 速览卡片中的核心数字（"812万""560万"） |
| 辅助 | 12px | 400 | 1.4 | 时间戳、补充说明、提示文字 |
| 标签 | 11px | 500 | 1.3 | 状态胶囊、标签徽章 |

**规则：**
- 全站只用两种字重：400（正文）和 500（标题和强调）。不用300（太轻看不清）、不用600/700（太重破坏从容感）。
- 所有文字用句首大写，不用全大写。中文不涉及此规则。
- 正文最大行宽：640px。超过640px的区域内容居中，不拉通全宽。对话流中Agent回复的文字遵守此限。
- 行高1.75是刻意的——比通常的1.5宽松。这是"呼吸感"的关键来源之一。Agent在跟你从容地讲话，不是急匆匆地塞信息。

## 第四章 间距与布局

### 4.1 间距刻度

```
 4px  —— 极紧：徽章内边距、图标与文字间距
 8px  —— 紧凑：列表项之间、表格行内边距
12px  —— 标准：卡片内边距（紧凑型）、按钮组间距
16px  —— 舒适：卡片内边距（标准型）、Agent组件间距
20px  —— 呼吸：对话流中消息之间的间距
24px  —— 分隔：大区块之间、Agent叙事中不同主题之间
32px  —— 段落：页面顶部留白、主要区域之间
```

**呼吸原则：** 对话流中的间距比传统界面更宽松。Agent的每段回复之间是20px，每个组件卡片之间是16px，卡片内部是12-16px。用户消息和Agent回复之间是24px。这比传统聊天应用（通常8-12px）宽松很多——因为这不是闲聊，是工作对话，每段信息都需要被认真阅读。

### 4.2 页面布局

```
┌──────────────────────────────────────────────────────────┐
│  顶部栏 · 高度 56px · 背景 #FFFFFF · 底部 0.5px 边框      │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│  导航栏   │  AI画布                                       │
│  宽 200px│  最大内容宽度 720px，水平居中                  │
│  背景     │  两侧自然留白                                 │
│  #F7F5F2 │                                               │
│  （与页面 │                                               │
│  底色同） │                                               │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘
```

**AI画布的内容宽度限制在720px并居中。** 这是关键设计决策——对话流不应该拉通整个屏幕宽度。720px大约是一本打开的书的宽度，人眼阅读最舒服。两侧的留白不是浪费空间，是**呼吸空间**。

导航栏与页面底色相同（`#F7F5F2`），不用深色背景。导航项之间用间距区分，不用分割线。当前激活项左边有一条2px的赭陶色竖线。

### 4.3 导航栏样式

```
┌──────────┐
│           │
│  ┃ 工作台  │  ← 当前项：左边2px赭陶色竖线，文字颜色变深
│    库房    │  ← 非当前项：泥土色文字
│    采购    │
│    呆滞    │
│    驾驶舱  │
│           │
│  ────────  │  ← 0.5px分割线，极淡
│    盘点    │
│    台账    │
│           │
└──────────┘
```

导航项用文字，不用图标。文字比图标信息量更大、视觉噪音更小。

## 第五章 组件样式

### 5.1 卡片（Card）

对话流中Agent渲染的数据卡片、操作卡片。

```
背景：#FFFFFF（亚麻白）
边框：无
圆角：12px
阴影：0 1px 3px rgba(26, 26, 30, 0.06), 0 1px 2px rgba(26, 26, 30, 0.04)
      ↑ 极淡的暖色阴影，让卡片从底色上微微浮起
内边距：16px
```

**卡片内的信息层次：**
- 顶部：区块标题（16px/500）——如"领用方案""消耗趋势"
- 中部：正文内容（14px/400）——分析文字、数据列表
- 底部：操作按钮（如果有）

**卡片不用边框。** 用微妙的阴影让卡片从Agent回复区域的`#F2F0EC`底色上浮起。这比0.5px边框更现代、更柔和。

### 5.2 按钮

| 类型 | 样式 | 用途 | 每页限制 |
|---|---|---|---|
| 主按钮 | 背景`#D4714A`（赭陶色），文字`#FFFFFF`，圆角8px | 每个对话流中最重要的一个操作——如"确认领用""批准" | 最多1个 |
| 次按钮 | 背景透明，边框0.5px `#A09A94`，文字`#6B6560`，圆角8px | 备选操作——如"选择备选""驳回" | 不限 |
| 幽灵按钮 | 无背景无边框，文字`#6B6560`，悬停时背景`#EFECEA` | 辅助操作——如"了解更多""展开详情" | 不限 |
| 快捷选项 | 背景`#EFECEA`，文字`#3D3832`，圆角全圆（胶囊形） | 点选式回答——如[故障维修] [保养更换] [技改] | 一组最多5个 |

**按钮高度统一36px。** 所有按钮，无论类型，高度都是36px。这是视觉节奏一致性的保证。

### 5.3 状态指示

| 元素 | 样式 | 用途 |
|---|---|---|
| 状态圆点 | 8px圆点，信号色 | 表格行前缀——绿点=充足、黄点=临界、红点=不足、紫点=呆滞 |
| 状态胶囊 | 信号底色背景 + 信号文字色，圆角全圆，11px文字 | 卡片内的状态标签——"✅充足""⚠️临界""🔴不足""🟣呆滞" |
| 水位条 | 高度6px，圆角3px，信号色填充 | 库存水位可视化——填充比例=当前/安全库存 |

**不用emoji作为状态指示。** ✅⚠️🔴🟣 在设计稿中用于说明，实际产品中用信号色圆点或胶囊替代。Emoji在不同系统上渲染不一致。

### 5.4 图表

图表嵌入在Agent回复的卡片中。配色遵循整体色系。

**柱状图/条形图：**
- 主色柱：`#D4714A`（赭陶色）
- 对比柱：`#A09A94`（亚麻灰）
- 异常标注柱：`#9B2C2C`（茜红文字色）
- 坐标轴和网格线：`#EFECEA`（浅麻色），0.5px
- 数据标签：`#6B6560`（泥土色），12px

**折线图：**
- 主线：`#D4714A`（赭陶色），2px
- 对比线/目标线：`#A09A94`（亚麻灰），1px虚线
- 数据点：4px圆点
- 面积填充：赭陶色10%透明度

**堆叠条形图（库存结构）：**
- 活库存：`#2D6A2D`（鼠尾草绿文字色）
- 安全储备：`#1A56A0`（靛蓝文字色）
- 可优化：`#8B5E0A`（赭黄文字色）
- 呆滞：`#6B4DC4`（紫藤文字色）

### 5.5 嵌入式表格

在对话流中展开的表格，不是传统的满屏数据表。

```
背景：#FFFFFF
圆角：12px（外框）
表头：背景 #F7F5F2，文字 #6B6560（泥土色），12px/500
数据行：文字 #1A1A1E（胡桃色），14px/400
数据行悬停：背景 #F7F5F2
行间分割：无线条，用交替底色（#FFFFFF 和 #FAFAF8）替代
数字列：右对齐，500字重
默认显示：5行 + "加载更多"按钮
最大宽度：跟随AI画布内容宽度（720px）
```

**不用传统表格的深色表头和粗边框。** 表格应该感觉像"Agent在对话中列了一张清单"，而不是"打开了一个独立的数据库视图"。

### 5.6 提醒横幅

Agent叙事中的提醒信息。不是弹窗，是嵌入在对话流中的静态横幅。

```
圆角：8px
左边框：3px，信号色
背景：信号底色
内边距：12px 16px
文字：14px/400，信号文字色
```

示例：
```
┌ 3px赭黄左边框 ─────────────────────────────────────┐
│  ⚠️ 3号线电机本月第4次更换轴承，频次异常。          │  ← 赭黄底色
│  建议排查电机底座对中和润滑状况。                    │     赭黄文字色
└────────────────────────────────────────────────────┘
```

## 第六章 对话流视觉节奏

### 6.1 对话流结构

整个AI画布是一个自上而下的对话流。视觉节奏如下：

```
┌──────────────────────── 720px AI画布 ──────────────────────┐
│                                                             │
│  [Agent主动叙事区域]                                        │
│  背景：#F2F0EC（Agent底色）                                 │
│  左侧：✦ 符号（赭陶色）                                    │
│  内部：文字段落 + 嵌入组件卡片（白色浮起）                  │
│                                                             │
│  ── 24px间距 ──                                             │
│                                                             │
│  [用户消息]                                                 │
│  背景：#FFFFFF，右对齐                                      │
│  圆角：12px                                                 │
│  最大宽度：480px（不超过画布的2/3）                         │
│                                                             │
│  ── 24px间距 ──                                             │
│                                                             │
│  [Agent回复区域]                                            │
│  背景：#F2F0EC                                              │
│  ✦ 前缀                                                    │
│  内部：                                                     │
│    文字段落（16px间距）                                     │
│    提醒横幅（16px间距）                                     │
│    数据卡片（16px间距，白色浮起）                           │
│    操作按钮组（16px间距，底部对齐）                         │
│                                                             │
│  ── 24px间距 ──                                             │
│                                                             │
│  [输入区域]                                                 │
│  固定在底部                                                 │
│  背景：#FFFFFF                                              │
│  上方：快捷选项胶囊按钮行                                  │
│  输入框：圆角12px，0.5px边框                                │
│  发送按钮：赭陶色圆形                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 Agent回复的渲染节奏

Agent的回复不是一次性出现，而是有节奏地**流式渲染**：

1. 先出 ✦ 符号和第一段文字（打字机效果，逐字出现）
2. 文字段落完成后，200ms 延迟
3. 组件卡片淡入出现（opacity 0→1，250ms，ease-out）
4. 如果有多个卡片，依次出现，每个间隔150ms
5. 操作按钮最后出现

这个节奏让用户感觉Agent在**思考后逐步展示**，而不是一股脑倾倒所有信息。

### 6.3 组件卡片的出现动画

```css
/* 卡片出现动画 */
@keyframes cardAppear {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.agent-card {
  animation: cardAppear 250ms ease-out forwards;
}
```

卡片从下方8px处微微上浮到正确位置，同时从透明变为不透明。动画极克制——只有8px的位移和250ms的时长。不要弹跳、不要旋转、不要缩放。

## 第七章 导航与页面切换

### 7.1 导航切换效果

点击导航项后：

1. 导航项左边出现2px赭陶色竖线（150ms渐入）
2. AI画布内容区域做一次轻微的淡入切换（opacity 150ms）
3. 新Agent的初始叙事从上方开始渲染

**不用页面滑动、不用侧边弹出、不用路由切换动画。** 切换是即时的+轻微淡入。快速、安静。

## 第八章 深色模式（预留）

当前版本不实现深色模式，但预留色值映射。Weave设计语言已定义了完整的深色方案：

| 浅色 | 深色 | 元素 |
|---|---|---|
| `#F7F5F2`（生麻色） | `#1A1A1E`（胡桃炭） | 页面底色 |
| `#FFFFFF`（亚麻白） | `#232328` | 卡片、输入框 |
| `#F2F0EC`（Agent底色） | `#1E1E23` | Agent回复区域 |
| `#1A1A1E`（胡桃色） | `#E8E6E3` | 主文字 |
| `#D4714A`（赭陶色） | `#D4714A`（不变） | 品牌点缀 |

**赭陶色在深色模式下不变**——它是贯穿两套色调的品牌线索。

## 第十章 设计检查清单

每个页面在交付前检查以下项目：

### 温度检查
- [ ] 所有灰色都是暖灰（黄调），不是蓝灰？
- [ ] 没有纯黑 `#000000` 文字？
- [ ] 没有纯白 `#FFFFFF` 用作页面背景？
- [ ] 截图去色后整体感觉温暖？

### 呼吸检查
- [ ] 对话流中消息间距 ≥ 20px？
- [ ] 正文行高 1.75？
- [ ] AI画布内容宽度 ≤ 720px？
- [ ] 组件卡片之间有 16px 间距？

### 从容检查
- [ ] 全站只用 400 和 500 两种字重？
- [ ] 每个视口最多1个赭陶色主按钮？
- [ ] 每张卡片最多2种信号色？
- [ ] 没有全大写文字？
- [ ] 提醒横幅使用平静的陈述语气？

### Agent区分检查
- [ ] Agent回复区域背景是 `#F2F0EC`（不是白色、不是蓝色）？
- [ ] ✦ 符号使用赭陶色？
- [ ] 组件卡片用白色背景从Agent区域浮起？
- [ ] 区分是微妙的（3%色差），不是强烈的？

### 动画检查
- [ ] 卡片出现动画：250ms ease-out，8px上浮？
- [ ] 没有弹跳、旋转、缩放动画？
- [ ] 页面切换是即时+轻微淡入，不是滑动？
- [ ] 流式文字有打字机效果？
